# CasinoKing Documentation Archive

Questa cartella contiene documenti storici, prompt, note di sessione, bug report chiusi e piani superati.

## Regola d'uso

I file archiviati non sono fonti primarie.

Usarli solo per capire il contesto storico di una decisione. Se un tema archiviato torna operativo, creare o aggiornare un documento attivo in `docs/` o `docs/md/` invece di modificare lo storico.

## Categorie

| Cartella | Contenuto |
| --- | --- |
| `bug-notes/` | Analisi bug o incidenti gia' chiusi o superati. |
| `prompts/` | Prompt operativi usati per revisioni precedenti. |
| `session-notes/` | Stato progetto, next step e analisi legate a sessioni passate. |
| `superseded-plans/` | Piani tecnici sostituiti da documenti piu' recenti o da codice gia' implementato. |

## Quando aggiungere nuovi file qui

Archiviare un documento quando:

- e' una nota di sessione superata
- e' un prompt usato per una revisione passata
- e' un bug report gia' chiuso
- e' un piano sostituito da un documento piu' recente
- contiene decisioni superate da fonti piu' recenti

Quando si archivia un documento, aggiornare anche `docs/README.md` se il file era citato come documento attivo.
